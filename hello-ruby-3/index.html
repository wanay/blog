Hello Ruby 3

Nov 8th, 2014 5:11 am

Ruby မှာ variable တွေရဲ့ data type တွေကိုကြိုတင်ကြေငြာပေးစရာမလိုပါဘူး။ဥပမာ

variable i ရဲ့ တန်ဖိုးကို 3 လို့ပေးလိုက်တာနဲ့ သူ့ရဲ့ type က integer type ဖြစ်သွားပါတယ်။

နောက်တစ်ခါ variable i ရဲ့ တန်ဖိုးကို “Hello World!” ဆိုတဲ့ String ထည့်လိုက်တာနဲ့ သူ့ရဲ့ type ကလည်း String type ဖြစ်သွားပါတယ်။

အခုတစ်ခါတော့ Decimal တန်ဖိုးပေးလိုက်တာနဲ့သူ့ရဲ့ type က float type ဖြစ်သွားပါတယ်။အဲဒါကြောင့် ruby မှာ variable တွေကို ဘာ data type ဖြစ်ပါတယ်ဆိုတာကိုကြေငြာပေးစရာမလိုပါဘူး။တစ်ခါတစ်လေမှာတော့ data type ကိုပြောင်းဖို့လိုအပ်လာတဲ့အခါမှာတော့ .to_i နဲ့ .to_u အစရှိသဖြင့့် method တွေကိုသုံးပါတယ်။အဲဒီလိုမျိုး Ruby မှာ built in methods တွေအများကြီးရှိပါသေးတယ်။ဥပမာအနေနဲ့

ကျွန်တော်တို့ “hello world” ဆိုတဲ့ string တစ်ခုကို variable i ထဲမှာသိုလှောင်လိုက်ပါတယ်။နောက်ပြီးတော့ upcase ဆိုတဲ့ method ကိုသုံးလိုက်တဲ့အခါမှာတော့ i ထဲမှာသိုလှောင်ထားတဲ့ string စာသားတွေအားလုံးကို Uppercase စာလုံးအကြီးတွေနဲ့ output ထုတ်ပြပေးပါတယ်။အဲလိုမျိုးပဲ string စာသားတွေအားလုံးကို Lowercase စာလုံးအသေးတွေပြောင်းချင်တဲ့အခါမှာတော့ downcase ဆိုတဲ့ method ကိုသုံးပါတယ်။​နောက်တစ်ခါ reverse method ကိုသုံးရင်တော့ string စာသားတွေကိုအနောက်ကနေအရှေ့ကိုပြောင်းပြန်ဖော်ပြပေးပါလိမ့်မယ်။တစ်ခြား method တွေစမ်းကြည့်ချင်ရင်တော့ i.methods လို့ရိုက်လိုက်တာနဲ့ method list တွေကိုထုတ်ပြ ပေးပါလိမ့်မည်။အပေါ်မှာနမူနာပြခဲ့တဲ့ variable တွေကို local variable လို့ခေါ်ပါတယ်။သူ့ကိုကြေငြာထားတဲ့ method အတွင်းမှာသာ ခေါ်သုံးလို့ရမှာပါ။တစ်ကယ်လို့ program ရဲ့ တစ်ခြားနေရာတွေကပါခေါ်သုံးမယ်ဆိုရင်တော့ global variable တစ်ခုအဖြစ်ကြေငြာထားလို့ရပါတယ်။global variable တွေကိုတော့ $ နဲ့သတ်မှတ်ပါတယ်။ဥပမာ $name တို့ $age ဆိုပြီး global variable အဖြစ်ကြေငြာနိုင်ပါတယ်။နောက်ထပ် instance variable နဲ့ class variable ဆိုတာတွေအကြောင်းကိုတော့ class တွေ object တွေနဲ့ getter/setter method တွေအကြောင်းရောက်မှ နားလည်နိုင်မှာကြောင့်ဒီနေရာမှာ မရှင်းပြတော့ပါဘူး။လောလောဆယ်တော့ instance variable တွေကို @ နဲ့စပါတယ်။ဥပမာ @description class variable တွေကို @@ နဲ့စပါတယ်။ဥပမာ @@name အဲဒီလောက်ပဲမှတ်ထားစေချင်ပါတယ်။ဒါပေမယ့် variable တွေကိုနာမည်ပေးတဲ့နေရာမှာ သတိထားစရာလေးတွေတော့ရှိပါတယ်။variable တွေကိုနာမည်ပေးတဲ့အခါ special character တွေပါလို့မရပါဘူး။ ချွင်းချက် အနေနဲ့ _ နဲ့တော့စပေးလို့ ရပါတယ်။နောက်တစ်ခါ variable နာမည်တွေမှာ number ပါလို့ရပါတယ်။ဒါပေမယ့် number တွေနဲ့တော့စပေးလို့ မရပါဘူး။နောက်ပြီး variable နာမည်တွေမှာ space တွေပါလို့မရပါဘူး။နောက်တစ်ခုကတော့ variable နာမည်အစမှာ စာလုံးအကြီးနဲ့ပေးလို့မရပါဘူး။ruby မှာ variable တွေ method တွေနာမည်ပေးတဲ့အခါမှာ camel case ထက် snake case ကိုသာအသုံးများကြပါတယ်။ကျွန်တော်အပေါ်မှာ variable တွေကိုကိန်းရှင် လို့ပြောခဲ့ပါတယ်။ဒါဆို ကိန်းသေ constant ရော မရှိဘူးလားဆိုရင် ရှိပါတယ်။ဥပမာအနေနဲ့ Pi ရဲ့တန်ဖိုးကို 3.142857142857143 လို့သတ်မှတ် လိုက်ပါမယ်။ထူးခြားတာက constant တွေကိုစာလုံးအကြီးနဲ့ စပေးရပါမယ်။

Pi ရဲ့တန်ဖိုးကို 3.142857142857143 လို့သတ်မှတ်ထားတဲ့အတွက် Pi ကိုခေါ်တိုင်း 142857142857143 ကိုပဲထုတ်ပေးပါမယ်။သူက variable တွေလို ကိန်းရှင်မဟုတ်တဲ့အတွက်တန်ဖိုးတွေကိုတိုးတာလျှော့တာတွေလုပ် လို့မရပါဘူး။Constant တွေကိုပြောင်းလို့မရတာတော့မဟုတ်ပါဘူး သူတို့ရဲ့တန်ဖိုးတွေကိုပြန်သတ်မှတ်လို့ရပါတယ်။
